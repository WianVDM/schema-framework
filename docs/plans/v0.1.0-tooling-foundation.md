```json
  "fixed": [],
  "linked": [],
  "access": "restricted",
  "baseBranch": "main",
  "updateInternalDependencies": "patch",
  "ignore": ["showcase"]
}
```

NOTE: `access: "restricted"` for private development. Change to `"public"` when publishing to npm.

[Testing]

### Validation Strategy

Since this milestone is tooling/configuration only, validation is manual:

1. **Changesets Workflow:**
   - Run `pnpm changeset` ŌĆö should prompt for change type and description
   - Run `pnpm changeset version` ŌĆö should bump versions and update CHANGELOGs
   - Verify `.changeset/` directory and `config.json` exist

2. **Build Verification:**
   - Run `pnpm build` ŌĆö must pass with zero errors
   - Run `pnpm typecheck` ŌĆö must pass with zero errors (if script exists)

3. **File Structure Cleanup:**
   - Verify all split files exist and export correctly
   - Verify all barrel `index.ts` files re-export correctly
   - Verify no import paths are broken (routes still import from same paths via barrel files)
   - Run `pnpm build` after split to confirm

4. **Hook Verification:**
   - Test PreToolUse hook detects core package modification and shows changeset reminder
